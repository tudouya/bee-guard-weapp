  4. 页面流程图

  4.1 有检测号流程

  检测主页 → 输入检测号+手机号 → 验证成功 → 蜂场调查问卷 → 邮寄说明 → 完成申请

  4.2 自费检测流程

  检测主页 → 选择套餐 → 扫码支付页面 → 用户扫码支付 → 支付凭证上传 → 审核状态页面 →
  审核通过获得检测号 → 蜂场调查问卷 → 邮寄说明 → 完成申请

  4.3 结果查询流程

  检测主页(结果查询Tab) → 手机号登录 → 检测记录列表 → 选择记录 → 结果详情 → 产品推荐

  5. 开发优先级

  Phase 1 - 核心申请流程 (P0):
  1. 改造检测主页：检测号验证入口 + 自费检测入口
  2. 开发扫码支付页面：显示二维码，保存功能
  3. 开发支付凭证上传页面：表单+图片上传
  4. 开发审核状态页面：状态跟踪
  5. 开发蜂场调查问卷页面：表单收集
  6. 开发邮寄说明页面：指导文档

  Phase 2 - 查询系统 (P1):
  7. 开发登录页面：手机号验证码登录
  8. 改造结果查询Tab：登录界面
  9. 开发检测记录列表：登录后显示
  10. 开发结果详情页面：详细结果+推荐

  Phase 3 - 功能完善 (P2):
  11. 产品推荐系统：企业产品vs平台推荐
  12. 消息通知：状态变更提醒
  13. 数据统计：检测历史统计
  14. 用户体验优化：加载状态、错误处理

  6. 技术要点

  6.1 关键功能实现

  - 二维码生成和显示
  - 图片选择、压缩、上传
  - 手机号验证码发送和验证
  - 支付状态跟踪和更新
  - 检测结果数据展示
  - 产品推荐逻辑

  6.2 数据存储

  - 用户登录状态管理
  - 检测记录本地缓存
  - 支付凭证临时存储
  - 问卷数据收集

  这个完整计划涵盖了所有必要页面和功能，确保了完整的业务闭环体验。
